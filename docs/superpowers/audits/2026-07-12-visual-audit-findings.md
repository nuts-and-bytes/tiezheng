# 铁证 IRONPROOF — 阶段 1 人工视觉验收 · 缺陷清单

**日期：** 2026-07-12
**方法：** 本地 `vite preview`（localhost:4173，与线上同一份 `dist/` 产物，origin 隔离，绝不碰线上真实数据）
**覆盖：** 零数据态 / 稀疏态（3 天）/ 丰富态（91 天 + 一个 30 组的怪物日）× 四个 Tab + 月度/年度海报
**运行时验证：** `read_console_messages` 零错误；`read_network_requests` 在海报重绘全程零请求（隐私铁律运行时成立）

## A. 真 bug（功能性，必须修）

### B1 · 力量趋势图在单点时全空 —— 主路径 bug
- **现象：** Y 轴刻度与 X 轴日期都画了，图区内一个像素的铁橙都没有（`getImageData` 像素扫描实锤）。
- **触发条件：** 某动作在当前范围内只有 1 个数据点。**App 默认范围是「周」**，所以任何一周内某动作只练过一次的用户（≈ 所有人）打开数据页，默认看到的就是空图。
- **表层根因：** `StatsScreen.tsx:275` `pointRadius: 0`（为求干净线条）→ 单点时既无线段可画、也无点可见；`StatsScreen.tsx:231` 的空态守卫只判断 `!weighted || !active`，没判断 `series.length < 2`。
- **深层根因（第一性原理）：** **信息架构错配**。顶部三个数字（打卡天数/总组数/总容量）是**周期汇总**，天然属于某个时间范围；力量趋势是**进步曲线**，天然属于全时段。把两者绑在同一个「周/月/年/全部」切换器上，必然产生"用户停在默认周 → 趋势图只剩 1 点 → 空白"。
- **修复方向：** 趋势图与周期切换器解耦（始终全时段 / 或最近 N 次）；同时 `pointRadius` 在点少时可见；`series.length < 2` 给降级态（显示该点数值 + "再练一次就有曲线"），而非空图表壳。

### B2 · 海报 footer 文字重叠 —— 成品级缺陷
- **现象：** 白色大字「你练过的，都有铁证。」与灰色小字「TIEZHENG.PAGES.DEV · 本地生成 · 照片不上传」**画在同一条基线上**，糊成一团。
- **影响：** 月度 + 年度**都中招**（共用 footer 绘制代码）。海报是要发出去的成品，糊字直接毁掉整张图。讽刺的是被压糊的正是「本地生成 · 照片不上传」这句隐私承诺。
- **根因：** Canvas 2D 两个 `fillText` 的 y 坐标相同或过近。

### B3 · 年度海报「THE YEAR IN IRON」溢出画布被裁
- **现象：** 大字 `2026` 右侧的英文标签冲出画布右边缘。
- **根因：** 英文标签的起始 x 跟着大字宽度推。`2026`（Anton 压缩体）远宽于月度的 `07`，于是被推出画布。**只在年度海报出现** —— 正是用户点名要的那个。

### B4 · 海报尺寸不固定，且比例非标准
- **实测：** 3 月 `1170 × 2853`（比例 0.410）、7 月 `1170 × 2715`（比例 0.431）。高度随内容（日历行数、训练日多寡）变化。
- **对比：** 标准 9:16 = 0.5625。当前是 ~1:2.4 的极瘦长条。
- **影响：** 每个月导出的海报尺寸都不一样；发朋友圈 / IG story 会被裁剪得乱七八糟。海报的核心用途就是"发出去"。

## B. 设计 / 一致性缺陷

| # | 缺陷 | 说明 |
|---|---|---|
| D1 | **零数据"硬摆零"不一致** | 日历顶部 `0 / 0 / 0`、我的页成就墙四个 `0`；而**同一个 app 里**数据页零态、PR 零态都做对了——换成引导文案（「还没有一条铁证。练一次，这里就会长出你的曲线。」）。同根因的一致性缺陷。 |
| D2 | **日历色块过饱和** | p90 归一**本身是对的**（已实测：30 组怪物日没有冲淡其余日子）。但满 alpha 的纯色块（红/蓝/紫）压在深色金属底上像儿童贴纸，与锻造工业风的克制感冲突；白色日期数字压在饱和红/紫上对比度不足。 |
| D3 | **部位均衡语义打架** | 横条统计**范围内**组数（如本周），右侧「已 37 天没练」却是**全时段**事实。同一行里两个时间语义。 |
| D4 | **周环比噪声** | `总容量 4.8t ↓88%` —— 因为上周撞上 30 组怪物日。容量的周环比天然噪声极大，↓88% 不代表用户练差了，但红色下降箭头的心理打击是实打实的。 |
| D5 | **emoji 视觉杂质** | 🔥（连续天数）、📷（拍照占位，甚至渲染成豆腐块方块）、⚒（铁龄）。整套 UI 是线性图标语言，emoji 是破绽。 |
| D6 | **年度热力图缺月份轴标签与图例** | 光秃秃一片色块，用户不知道哪列是几月。日历页有图例，这里没有。 |
| D7 | **日历上月溢出格：有色块但日期暗灰** | 6/29 练了背（蓝色块），日期数字仍是"非本月"的暗灰 —— 有色块却读不出日期。 |
| D8 | **PR 免责声明位置倒置** | 「按预估 1RM（Epley）排名」在列表**最底部**（要滚过 12 条才看到），而虚高的 `252 kg`（腿举 180kg×12 的 Epley 推算）在**最顶部**。免责声明放错了位置。 |
| D9 | **海报双入口文案不一致** | 数据页「生成训练海报」vs 我的页「导出训练海报」，副标题却完全相同。 |
| D10 | **引导页垂直留白过大** | 真机 390×844 下成立（非工装假象），4/4 屏最严重：内容只占顶部 1/4。 |
| D11 | **今日页设计卡组数冗余** | meta 行 `8 组 · 1,160 kg 容量` + 右栏大号 `8组`；无重量数据时 meta 退化成 `3 组`，与右栏完全重复。 |

## C. 已验证达成的用户诉求 ✅

- **品牌 logo 出现**：启动屏 / 今日页头 / 我的页 / 数据页零态 均有铁字钢印。
- **各部位有专属图标与颜色**：7 个部位（胸/肩/背/腿/手臂/核心/有氧）在引导、LogFlow 选部位、日历图例、部位均衡中一致使用。
- **日历"一眼看出练了哪个部位"**：色块 + 部位小图标；多部位日显示多图标。
- **p90 归一有效**：30 组怪物日**没有**把 5–8 组的普通日冲淡成灰。若用 `max()` 归一则会。
- **首次引导介绍了海报导出**：第 3 屏明确讲月度/年度海报，并标注「全本地生成 · 照片不上传」。
- **隐私铁律运行时成立**：海报生成全程零网络请求（实测）；「仅存本机」「照片不含在导出文件中」在 UI 可见。
- **数据充足时的可视化是好的**：「全部」范围下 e1RM 曲线（60→90 渐进）、年度热力马赛克、PR 榜 —— 都很扎实。
