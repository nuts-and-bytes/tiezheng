# 铁证 IRONPROOF — 产品设计规格

日期：2026-07-08
状态：已与产品负责人逐节确认

## 0. 一句话定位

面向国内健身爱好者的健身打卡 PWA：**你练了，证据呢？** 记录训练、体重与照片，把坚持变成看得见的曲线和可分享的「铁证海报」。

## 1. 背景与约束

- 前作：`nuts-and-bytes/fitness-daily`，单文件 HTML + LocalStorage + Chart.js 的自用 PWA。仅作参考，不复用代码。
- 目标用户：国内健身爱好者，微信链接传播，iOS 为第一优先平台。
- 成本约束：**零成本验证期**。托管 Cloudflare Pages 免费层，后端 Supabase 免费层（Postgres 500MB + Storage 1GB）。
- 已知风险：`*.pages.dev` 与 Supabase 在国内「大体可用、不保证稳定」。若传播受影响，最低成本补丁为购买域名（约 ¥60/年）接 Cloudflare，验证后再投入。
- 登录方式：邮箱 + 密码（Supabase Auth）。微信登录需企业资质、短信按条收费，验证期不做。

## 2. 产品铁律（所有功能决策的判据）

1. 打开到完成打卡 **≤ 10 秒、≤ 4 次点击**，超过就砍交互。
2. **任何情况下不丢一条记录**：弱网、断网、换手机都不丢。
3. 每条记录都要变成看得见的反馈：日历亮格、曲线上扬、海报素材。

## 3. 差异化优势（对外话术）

- 零广告、零弹窗、零课程推销，打开即是记录。
- 健身房地下室断网照样用（本地优先，永不转圈）。
- 拍照留证 + 季/半年/年度「铁证海报」：数据变成情绪出口与裂变入口。
- 微信链接直接用、免下载，添加到主屏即原生体验。
- 数据主权：一键导出 CSV/JSON，注销即删。

## 4. 品牌与视觉

- 名称：**铁证**（TIEZHENG · IRONPROOF）。双关「撸铁」+「铁证如山」。Slogan：「你练过的，都有铁证」。
- 视觉方向：**铁与暗**。纯黑底（#0A0A0B）、深灰卡片（#1C1C1E）、锻铁橙单强调色（#FF5C1F，渐变至 #FF8C42）、系统字体栈、大数字排版、高对比。Apple Fitness 式暗色原生感。
- Logo：待实现阶段单独设计（图形需在 180px 苹果图标下清晰可辨），主题延续「铁 + 证据」意象。
- 部位色（日历色点与图表统一）：胸=锻铁橙、肩=琥珀黄、背=钢蓝、腿=深紫、手臂=青绿、核心=玫红、有氧=灰绿。具体色值实现期在暗底上校准对比度。

## 5. 功能范围与分期（每期结束都是可用产品）

### Phase 1 · 单机完整版（无登录即完整可用）
- 训练记录：7 部位 → 动作 → 组数必填、重量/次数选填。
- 日历（月视图 + 日详情）、体重记录、全部图表。
- 拍照打卡（照片仅存本地）。
- 铁与暗 UI、Logo、PWA 添加到主屏。

### Phase 2 · 账号与云
- 邮箱注册/登录、云同步、照片云备份、换设备恢复。
- 首次登录将本地数据一键迁移至账号。

### Phase 3 · 传播与打磨
- 铁证海报（季/半年/年度）导出。
- 微信内打开引导层、首次使用 30 秒上手流程。
- 隐私政策页、注销删除。

### 明确不做（验证期）
- 微信登录、短信登录、App 内公开社区/动态流、训练课程内容、Android 原生打磨（能用即可）、多语言、磅/千克切换（仅 kg）。

## 6. 信息架构

4 个底部 Tab + 1 条全屏记录流：

- **Tab 1 今日**：日期问候、本周目标进度环（默认每周 4 练）、今日训练卡（未练 → 「+ 开始今日训练」；已练 → 摘要）、体重快捷录入、拍照打卡入口、连续/累计打卡（次要位置）。
- **Tab 2 日历**：月视图，每日部位色点，有照片日子带相机角标；点日期进**日详情**（动作列表 组×重量×次数、当日体重、照片，可编辑删除）。
- **Tab 3 数据**：部位频次雷达（30/90/365 天切换）、周频次柱状图（近 12 周 + 目标线）、体重趋势（原始点 + 7 日均线）、力量曲线（按动作，仅对填了重量的动作开放）、累计大数字（总打卡天数/总组数/总容量）、体型照片时间轴（私密）。
- **Tab 4 我的**：账号与同步状态、铁证海报入口、每周目标设置、动作库管理、数据导出 CSV/JSON、隐私政策/注销。

**记录流**（全屏模态，随时退出自动存草稿）：
① 选部位（7 宫格，可多选）→ ② 选动作（该部位动作库，最近使用置顶，可搜索/新建）→ ③ 记组数（± 步进器默认 3 组，每组可展开选填重量×次数）→ ④ 完成打卡（今日总结 + 日历亮格动画 + 顺手拍照）。
最快路径：4 次点击约 8 秒。

Tab 外页面：首次引导（选每周目标 → 直接进第一次打卡）、微信 UA 检测引导层（「右上角 → 在 Safari 打开」）。

**打卡指标设计**：主指标为「每周训练目标完成度」（避免连续天数断签劝退的损失厌恶），连续天数仅次要展示。

## 7. 数据模型

本地（IndexedDB/Dexie）与云（Supabase Postgres）同构。每行必带：`id`（本地生成 UUID v4）、`updated_at`（毫秒时间戳）、`deleted_at`（软删除，null=有效）。

| 表 | 字段要点 |
|---|---|
| `workouts` | `user_id?`, `date`(YYYY-MM-DD，每用户每天一条), `note?` |
| `workout_items` | `workout_id`, `exercise_id`, `order`, `sets` JSONB `[{weight?, reps?}, …]`（组数=数组长度） |
| `exercises` | `name`, `body_part`(7 枚举: chest/shoulder/back/leg/arm/core/cardio), `user_id?`（null=系统预置） |
| `weight_logs` | `date`(每用户每天一条), `weight_kg` |
| `photos` | `date`, 本地 blob / 云 `storage_path`, `size` |
| `profile` | `weekly_goal`(默认 4), `nickname?` |

预置动作库约 40 个（每部位 4–8 个常见动作，中文名，实现期定稿清单）。

## 8. 同步机制（Phase 2 启用，Phase 1 即按此结构落库）

- 写操作一律本地先行；每次写入进待同步队列（outbox）。
- 有网且已登录：批量 upsert 推送 + 按「上次同步时间之后更新的行」增量拉取。
- 冲突：比较 `updated_at`，**新者胜**（LWW）。删除即软删，同规则合并。
- 首次登录：本地数据全量打 `user_id` 上传（即迁移）。
- 失败重试：指数退避；「我的」页展示最后同步时间。

**安全**：所有表启用 RLS（`user_id = auth.uid()`）；照片桶私有，路径 `{user_id}/{date}.jpg`，读取走短时签名 URL。

## 9. 照片管道

拍照/选图 → Canvas 压缩（长边 1280px，JPEG q0.8，约 100–200KB）→ 存本地 IndexedDB → 登录后后台上传。每天 1 张上限。1GB 免费层约可支撑 6000 张，验证期够用；配额预警实现为客户端计数提示。

## 10. 铁证海报

入口：我的 → 选周期（季/半年/年）→ 离屏 Canvas 2x 绘制 1080 宽长图：
品牌头 → 三个核心大数字（打卡天数/总组数/总容量）→ 部位分布图 → 迷你热力日历 → 体重变化 → 可选勾选 1–3 张照片（默认不含）→ slogan +产品链接二维码（裂变入口）。
导出：优先系统分享面板（Web Share API），降级为长按保存。

## 11. iOS 伪原生清单

- manifest `standalone`、theme_color #0A0A0B、180px apple-touch-icon、启动屏图。
- `viewport-fit=cover` + safe-area 内边距（灵动岛/底部横条）。
- 禁双击缩放、去点击高光、`100dvh`、`overscroll-behavior` 防橡皮筋穿透。
- 系统字体栈；按钮按压态微动画（补偿无震动反馈）。
- Service Worker 预缓存 app shell；新版本顶部提示「点击更新」。
- iOS Safari 7 天不活跃清数据：添加到主屏可基本豁免，云同步为最终兜底（账号功能的核心卖点话术）。

## 12. 错误处理

- 网络失败不阻塞任何记录操作（本地先行 + 队列重试）。
- 表单硬校验：体重 20–300kg、组数 1–20、重量 0–500kg、次数 1–100。
- 照片压缩失败/超限有明确中文提示；Supabase 认证错误中文化映射。
- 全局 React 错误边界，本地环形日志便于排查。

## 13. 测试策略

- **Vitest 单测**（核心纯逻辑）：同步合并（LWW/软删）、统计聚合（各图表数据变换）、海报数据组装、表单校验。
- **组件测试**（少量）：记录流关键交互（Testing Library）。
- **手动清单**（入库 `docs/checklists/ios-device.md`）：iOS 真机添加主屏、断网打卡、拍照、海报保存、微信内引导。
- **CI**：GitHub Actions lint + test + build；主分支自动部署 Cloudflare Pages。

## 14. 技术栈

Vite + React + TypeScript + Tailwind 4 + Zustand + Dexie + Chart.js（按需注册）+ vite-plugin-pwa（Workbox）+ Supabase JS。

## 15. 验收标准（产品级）

1. iPhone Safari 添加到主屏后：全屏无浏览器框、启动 < 2s（二次启动走缓存）、安全区无遮挡。
2. 飞行模式下完成一次完整打卡（部位+动作+组数+拍照），恢复网络后（Phase 2）自动同步成功。
3. 新用户从打开到完成第一次打卡 ≤ 60 秒（含引导）。
4. 日历/雷达/体重/力量四类图在 90 天量级数据下渲染流畅。
5. 海报在 iPhone 上一次点击生成并可保存/分享，图片清晰（2x）。
